🧠 Qu'est-ce que ReAct ?

ReAct = Reasoning + Acting

Un paradigme où l'IA alterne entre réflexion (pensée structurée) et action (utilisation d'outils) pour résoudre des tâches complexes.

💭 Thought
Raisonnement
→
🔧 Action
Exécution
→
👁️ Observation
Résultat
  • Transparence : On voit le raisonnement étape par étape
  • Correction d'erreurs : L'agent peut se rendre compte de ses erreurs et ajuster
  • Accès aux outils : Connexion à des APIs, bases de données, calculatrices
  • Moins d'hallucinations : Basé sur des faits observés, pas juste sur l'entraînement

🔗 Chain of Thought (CoT) Détaillé

Le CoT force le modèle à décomposer son raisonnement de manière explicite :

Question: "Quel est le meilleur restaurant italien près de moi 
          qui accepte les cartes de crédit et a une note > 4.5?"

Thought 1: Je dois d'abord trouver ma localisation actuelle
Action: get_user_location()
Observation: "Paris, 11ème arrondissement"

Thought 2: Maintenant je cherche les restaurants italiens dans cette zone
Action: search_restaurants(cuisine="italian", location="Paris 11")
Observation: [Liste de 5 restaurants]

Thought 3: Je dois filtrer ceux qui acceptent les cartes et ont note > 4.5
Thought 4: Le restaurant "Luigi's" correspond: note 4.7, accepte CB
Final Answer: "Luigi's, 4.7★, 123 rue de la Roquette"
💡 Astuce : Ajoutez "Pense étape par étape" dans votre prompt pour activer le CoT automatiquement.

• Variables temporaires

📋 Pseudo-code du Cycle

def agent_loop(user_input, max_iterations=10):
    context = retrieve_relevant_docs(user_input)
    memory = load_conversation_history()
    
    for i in range(max_iterations):
        # Étape 1: Réflexion
        prompt = build_prompt(user_input, context, memory)
        llm_output = llm.generate(prompt)
        
        thought, action, action_input = parse_react_output(llm_output)
        memory.add_thought(thought)
        
        # Étape 2: Vérification humaine si nécessaire
        if requires_human_approval(action):
            if not human_approval(action, action_input):
                continue  # L'agent doit trouver une autre solution
        
        # Étape 3: Exécution
        observation = execute_tool(action, action_input)
        memory.add_observation(observation)
        
        # Étape 4: Vérification de la réponse finale
        if is_final_answer(thought):
            # Évaluation humaine optionnelle
            if requires_human_validation(thought):
                final_answer = human_review(thought)
            else:
                final_answer = thought
            
            save_to_long_term_memory(user_input, final_answer)
            return final_answer
    
    raise Exception("Max iterations reached")

👤 Human-in-the-Loop (HITL)

L'humain intervient aux points critiques pour valider, corriger ou guider l'agent :

🚦 Quand activer le HITL ?

  • Confiance basse : Score < 0.7
  • Actions irréversibles : Suppression, envoi d'argent
  • Données sensibles : Santé, juridique, financier
  • Ambiguïté : Plusieurs interprétations possibles
  • Nouveauté : Cas jamais vu auparavant

💡 Modes d'intervention

  • Approval : "Peux-tu confirmer cette action ?"
  • Correction : L'humain modifie la proposition
  • Guidance : "Essaie plutôt cette approche..."
  • Escalade : Transfert à un expert humain
# Configuration des règles HITL
HITL_RULES = {
    "financial_transaction": {
        "threshold": 1000,  # Euros
        "require_approval": True,
        "escalate_if": "amount > 10000"
    },
    "medical_advice": {
        "always_require_human": True,
        "disclaimer": "Ceci ne remplace pas un avis médical"
    },
    "data_deletion": {
        "require_confirmation": True,
        "cooling_period": "24h"  # Délai de rétractation
    }
}

def check_hitl_required(action, context):
    if action.type in HITL_RULES:
        rule = HITL_RULES[action.type]
        if rule.get("always_require_human"):
            return True
        if action.amount > rule.get("threshold", float('inf')):
            return True
    return False

💻 Exemple Complet : Agent Support Client

Cas d'usage : Gestion de réclamations avec validation humaine

from langchain.agents import AgentExecutor, create_react_agent
from langchain.memory import ConversationBufferWindowMemory
from langchain.tools import Tool
from langchain.vectorstores import Chroma
from langchain.embeddings import OpenAIEmbeddings

# 1. Configuration Mémoire
memory = ConversationBufferWindowMemory(
    k=10,  # 10 derniers échanges
    return_messages=True,
    memory_key="chat_history"
)

# 2. Base de connaissances (RAG)
embeddings = OpenAIEmbeddings()
vectorstore = Chroma(
    persist_directory="./docs",
    embedding_function=embeddings
)
retriever = vectorstore.as_retriever(search_kwargs={"k": 3})

# 3. Définition des Outils
def check_order_status(order_id: str) -> str:
    """Vérifie le statut d'une commande"""
    # Appel API base de données
    return db.query(f"SELECT status FROM orders WHERE id={order_id}")

def process_refund(order_id: str, amount: float, reason: str) -> dict:
    """Traite un remboursement (nécessite validation humaine si > 100€)"""
    if amount > 100:
        return {"status": "pending_approval", "requires_human": True}
    return {"status": "processed", "transaction_id": generate_id()}

def search_knowledge_base(query: str) -> str:
    """Recherche dans la base de connaissances"""
    docs = retriever.get_relevant_documents(query)
    return "\n".join([d.page_content for d in docs])

tools = [
    Tool(
        name="OrderStatus",
        func=check_order_status,
        description="Vérifie le statut d'une commande. Input: order_id"
    ),
    Tool(
        name="ProcessRefund",
        func=process_refund,
        description="Traite un remboursement. Input: order_id, amount, reason"
    ),
    Tool(
        name="KnowledgeBase",
        func=search_knowledge_base,
        description="Recherche des informations dans la documentation"
    )
]

# 4. Prompt ReAct personnalisé
REACT_PROMPT = """Tu es un assistant support client expert.
Utilise le format ReAct suivant:

Question: la question de l'utilisateur
Thought: tu dois toujours réfléchir étape par étape
Action: l'action à effectuer (parmi {tool_names})
Action Input: les paramètres de l'action
Observation: le résultat de l'action
... (ce processus Thought/Action/Observation peut se répéter N fois)
Thought: Je connais maintenant la réponse finale
Final Answer: la réponse finale à l'utilisateur

Si l'action nécessite une validation humaine (remboursement > 100€, 
modification de données sensibles), indique-le clairement.

Contexte: {context}
Chat History: {chat_history}
Question: {input}
{agent_scratchpad}"""

# 5. Création de l'agent
agent = create_react_agent(llm, tools, prompt=REACT_PROMPT)
agent_executor = AgentExecutor(
    agent=agent,
    tools=tools,
    memory=memory,
    verbose=True,
    max_iterations=15,
    handle_parsing_errors=True
)

# 6. Fonction d'exécution avec HITL
def execute_with_hitl(user_input: str) -> str:
    response = agent_executor.invoke({"input": user_input})
    
    # Vérification post-exécution
    if "pending_approval" in str(response):
        # Notification à l'équipe humaine
        notify_human_team(response)
        return "Votre demande a été transmise à notre équipe pour validation. Vous recevrez une réponse sous 24h."
    
    # Logging pour évaluation
    log_interaction(user_input, response, metadata={
        "timestamp": datetime.now(),
        "memory_size": len(memory.buffer),
        "tools_used": extract_tools_used(response)
    })
    
    return response["output"]

# 7. Exemple d'utilisation
result = execute_with_hitl(
    "Je veux retourner ma commande #12345 et être remboursé de 150€"
)
print(result)

🎬 Scénario Temps Réel

Exécution complète d'une requête complexe :

📞 Requête Utilisateur

"Bonjour, j'ai reçu un produit défectueux (commande #78945). Je voudrais un remboursement et connaître votre politique de retour pour les produits électroniques."

🤖 Thought 1: L'utilisateur demande un remboursement pour la commande #78945 et des infos sur la politique de retour. Je dois d'abord vérifier le statut de la commande.
🔧 Action 1: OrderStatus
Action Input: {"order_id": "78945"}
👁️ Observation 1: {"status": "delivered", "date": "2024-01-15", "product": "Laptop Dell XPS 15", "amount": 1299.99, "within_return_window": true}
🤖 Thought 2: La commande est livrée depuis moins de 30 jours, donc éligible au retour. Le montant est de 1299.99€ (> 100€), donc nécessite validation humaine. Je dois aussi chercher la politique de retour.
🔧 Action 2: KnowledgeBase
Action Input: "politique retour produits électroniques"
👁️ Observation 2: "Retour possible sous 30 jours. Produit doit être dans son emballage d'origine. Remboursement sous 5-10 jours ouvrés après réception."
Action 3: ProcessRefund
Action Input: {"order_id": "78945", "amount": 1299.99, "reason": "produit défectueux"}
👁️ Observation 3: {"status": "pending_approval", "requires_human": true, "ticket_id": "REF-2024-5678"}
🤖 Thought 3: Le remboursement nécessite une validation humaine car > 100€. J'ai créé le ticket REF-2024-5678. Je dois informer l'utilisateur et donner les instructions de retour.
✅ Final Answer:
"J'ai bien pris en compte votre demande pour la commande #78945 (Laptop Dell XPS 15).

✓ Votre commande est éligible au retour (moins de 30 jours)
✓ Un ticket de remboursement a été créé: REF-2024-5678
⏳ Montant: 1299.99€ - Validation requise (délai: 24-48h)

📦 Instructions de retour:
- Emballez le produit dans son emballage d'origine
- Imprimez l'étiquette de retour (email envoyé)
- Déposez le colis en point relais

💰 Une fois validé par notre équipe, le remboursement sera effectif sous 5-10 jours ouvrés."
👤 Intervention humaine : L'équipe support reçoit une notification pour valider le remboursement de 1299.99€. Elle vérifie l'historique client et approuve sous 2h.

📋 Bonnes Pratiques & Pièges à Éviter

✅ À Faire

  • Timeouts : Limitez à 10-15 itérations max
  • Logging complet : Tracez chaque Thought/Action
  • Gestion d'erreurs : Catch les exceptions d'outils
  • Rate limiting : Évitez les appels API excessifs
  • Validation input : Sanitize les entrées utilisateur
  • Tests réguliers : Scénarios edge cases
  • Documentation : Décrivez chaque outil précisément

❌ À Éviter

  • Boucles infinies : L'agent qui tourne en rond
  • Context overflow : Trop d'historique = coût + erreurs
  • Outils redondants : Gardez une liste minimaliste
  • Prompt trop long : Restez concis dans les instructions
  • Ignorer la confidentialité : Ne loggez pas les données sensibles
  • Sauter le HITL : Même si "ça marche" sans
  • Pas de monitoring : Surveillez les métriques en prod
